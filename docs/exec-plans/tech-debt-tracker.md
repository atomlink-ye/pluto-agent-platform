# Tech Debt Tracker

## Purpose

This document tracks deferred work that should remain visible without pretending it is part of the minimum stable core.

## Current debt themes

### Deferred platform breadth

- multi-surface parity beyond the primary operator surface
- advanced orchestration modes beyond the simplest governed path
- advanced analytics and reporting

### Deferred platform hardening

- richer tenancy strategy
- deeper access control and RBAC
- more advanced policy enforcement paths

### Deferred product richness

- broader playbook libraries
- richer artifact lineage views
- more advanced replay and investigation tools

## Usage rule

Items here should remain visible until either:

- they are promoted into an active execution plan, or
- they are explicitly dropped as non-goals
