# Legacy Agent CoWork Summary

## Purpose

Summarize what the older Agent CoWork codebase contributes as historical reference.

## What it contributed

The legacy project validated several important product ideas:

- long-running governed agent execution
- visible workflow or run progress
- mailbox- and artifact-oriented collaboration ideas
- the importance of review and evaluation loops
- strong documentation discipline around architecture and product semantics

## What is retained conceptually

- operator-visible execution
- approvals and artifacts as product concepts
- multi-role coordination as a first-class concern
- a strong documentation-first design culture

## What is not being retained as the long-term implementation base

- the Python-first system architecture
- the prior split between the product and external runtime as the primary long-term shape
- the older vocabulary where workflow objects dominated the model too strongly

## How to use this reference

Use the legacy project as a source of product semantics and lessons learned, not as the implementation blueprint for this repository.
